Responsive Redesign Hand-In

Overview

The goal of this project was to redesign an existing website to make it more usable, accessible and responsive across various screen sizes (computer, tablet and mobile). I engaged in the design process, from sketching to hi-fi mockups, and then used HTML and CSS to code the interface!

Part 1: Identifying Usability Problems

Picking a Webpage

This website provides people with information about Columbia Road in London, which is often bustling on Sundays with a flower market and many local shops. I chose the Columbia Road website because I feel that the existing interface lacks visual hierarchy and is not intuitive to use, which is unfortunate given the vibrant energy in this part of London. Usability Problems

  • The navigation menu only appears at the beginning of the webpage, yet the logo and timing remain at the top of the screen when scrolling. This makes it tedious to navigate to another page since you must scroll to the beginnning of the website, and it contradicts typical website conventions in which the navigation menu constantly remains on the screen.
  • It is unclear what text and images are clickable since they looks visually the same as the rest of the paragraphs and images.
  • There is little visual distinction between different sections, particularly immediately following the initial image. Also, since the font weights and sizing are inconsistent in this part of the website, it is difficult to find information quickly.
  • When resizing the interface, one of the shops disappears, yet there is a space in its place. This can be confusing to users as to why there is a discrepancy between different screen sizes.

Accessibility

The problems detected by WebAIM Wave, an accessibility checker, include a linked image missing alt text, empty links and redundant links. I agree with the issues detected. Some of the links appear to be hidden behind graphic elements, which may be confusing to screenreaders (users of the website without screenreaders would not notice the links). In addition, when resizing the interface to tablet or mobile, one of the shop images dissapears yet the link remains. Overall, there appeared to be no accessibility issues with the color contrast, which makes sense given that the black/dark gray text is presented on a light background.

Part 3: Responsive Redesign

You can visit my redesigned website here!

WebAIM Wave detected zero errors with the website and had one alert about a skipped heading size. This decision was intentional because I wanted the "Welcome to Columbia Road" to be the focus when opening the page, and the website felt too crowded in tablet/mobile version when the opening times was the next heading size. This project taught me a lot about the design process, and I hope to continue using these practices when creating websites in the future.
